Cisco Programmability for Nexus 9000 in NX-OS Standalone Mode

Offering Formats

This course is offered in the following formats:

  • Live, Instructor-Led Training (schedule)
  • Self-Paced, Online Training (coming soon!)

Course Overview

In this 2-day course, you will learn how to begin programming and automating Cisco Nexus 9000 switches. You will become experienced with the the implementation of Cisco’s Open NX-OS architecture, with Data Management Engines and Managed Object Models, and ReSTful APIs. You will learn how open source tools are changing the way businesses interact with their network infrastructure every day, and why we as network engineers must change as well. In this course you will learn what everyone means when they talk about “Infrastructure as Code”.

It is designed for customers, network engineers, systems engineers, and field engineers who wanting to learn how to accelerate their deployments, minimize human error and simplify and enhance every aspect of their network infrastructure with programmatic approaches.

Discussion will also be presented on how to bring about a change to a true ‘DevOps’ culture in your company.

Who Should Attend?

  • Network Administrators
  • System Engineers
  • Solution Architects
  • System Integrators

Recommended Prerequisites

  • Basic understanding of networking
  • Basic understanding of Nexus switches
  • Basic understanding of Linux (recommended, but not required)

Course Agenda

Programmability for Nexus 9K Agenda – Day 1
Lecture 01 Nexus 9000 Open NX-OS Architecture
Students will learn how the Cisco Nexus 9000 is really a hardened, yet open Linux device, allowing them access to familiar Linux Containers (LXC), Bourne Again Shell (BASH) and the ability with access to install 3rd party applications with YUM.
Lab 01 Learning To Work with Nexus 9000 as a Linux Device
Students put into practice the basics of working with Linux: Navigating the file system, working with basic utilities such as grep, vi, ifconfig, tcpdump, and even starting your own scripts at boot time with init.d, installing apps with yum, and learning about the vShell and how to interact with the 9000 switch from Bash.
Lecture 02 Fundamental Programmability Concepts
Students begin to see the traditional CLI from a new perspective. They learn how the CLI has not gone away, but become much richer in power and capabilities. They learn how the NX-API brings new power, scalability and error reporting, as well as structured feedback for simple commands, and they learn how to use the NX-API Sandbox to learn by simply playing.
LUNCH BREAK
Lab 02 Explore Basic Interaction with Nexus API Interfaces
Students take the NX-API out for a spin to understand with hands-on interaction about what the different types of messaging and command-line formats are, how they interact with one another, and what different types of input they take and output they provide. Learners also run pre-built applications to see how much there is to offer from using programmatic API interfaces with Nexus 9000.
Lecture 03 Model Driven Programming (ReST API for Nexus 9000)
Students come to understand what ReST is, what a DME and a Managed Object Model are, and why they will learn to love them. The instructor will begin to demonstrate what both Parent-Child objects are in an Object Store, the difference between a Name, a Relative Name and a Distinguished Name, as well as what Attribute-Value pairs are and how serialized structured data will change their life, once they leap over the learning hurdle and embrace it. Students also learn about the Object Browser – Visore, and how to maximize their output using it.
Lab 03 Explore ReST on Nexus 9000 Using Postman
Students learn how to begin interacting with the Nexus 9000 switch using a ReST client such as the Google Chrome plugin called Postman. Students will learn to login to the switch, capture a token used for passing subsequent ReST calls in an authenticated manner, and begin making basic API calls and returning structured MO output. Students also take a journey through Visore.
DevOps / Programmability Agenda – Day 2
Lecture 04 Automating Configuration Management
Students will learn how Cisco POAP and Cisco Ignite work. They will also learn the basics of Agent-based vs Agent-less configuration management systems such as Puppet and Chef vs Ansible. They will learn what a YAML file is and how to structure the data inside. They will also learn the difference between the Imperative vs Declarative methods of Management and what the basis of Promise Theory is.
Lab 04 Power-On Auto Provisioning (POAP) and Cisco Ignite
Students learn how to setup the required DHCP and TFTP servers needed for POAP, and how to interact with a multi-type switch environment (Nexus 9K + 7K/5K). Students will erase their N9K switches completely and boot via POAP. They will then continue on to setup the broader framework of Cisco Ignite.
Lab 05 Configuring Ansible (with an Intro to Chef & Puppet)
Students will install and setup an Ansible server, along with a Playbook of YAML Plays, and provision all of their N9K switches with VXLAN w/ EVPN control plane using these Plays. They will also take a brief look at installing the Chef and Puppet agents inside the OS and discus what would be needed to support these.
LUNCH BREAK
Lecture 05 Applying Network Programmability with Open Applications
Students go much deeper in beginning to learn how so many common tools that developers use every day can begin to transform the way they think about, see and interact with their network infrastructure. They learn what Version Control Systems are and how things like Git and Github can simplify their life. They learn about Jenkins and Maven and what CI/CD is and why they might want to consider using it vs traditional Waterfall approaches to configuration management. They learn about Splunk and TCollector and how these all with change the way they do things in the Data Center.
Lab 06 Working with Open Source Tools on Nexus 9000
Students download and install git, splunk forwarder and tcollector. They create a simple script that performs a git commit every time configuration is changed on the switch. They begin exporting data to Splunk and OpenTSB.
Lecture 07 Cisco DevNet, OpenNXOS.cisco.com and VIRL for Next Steps
Students will wrap up with things they can takeaway to immediately continue learning to interact with Infrastructure as Code in their daily lives, in safe, sandbox environments. They will learn where documentation is, where open source developers commit public code, and how to test everything in a simulated environment.
