Cisco ACI Fundamentals for Network Engineers

Offering Formats

This course is offered in the following formats:

  • Live, Instructor-Led Training (schedule)
  • Self-Paced, Online Training (samples)

Course Overview

In this 5-day Cisco ACI for Network Engineers course, you will become fully adept at deploying, provisioning and maintaining a Cisco® Application Centric Infrastructure (ACI), including the planning, design, and implementation of ACI.

It is designed for Architects, Systems Engineers and Field Engineers who are selling, implementing and/or managing Cisco ACI. The course dives deep into each of the key concepts behind ACI’s application-centric infrastructure and policy-based architecture. You will spend each day listening to lecture, then configuring most every aspect of the ACI solution, including configuring tenants, application network profiles (ANP), contracts, endpoints, VMM integration and how to connect the ACI Fabric to an external L2 and L3 networks, along with L4-L7 services.  Migration strategies for existing applications to an ACI fabric will be discussed and automation of APIC REST API through scripting (JSON/XML). The course will include intense hands-on labs on real physical hardware of all of the key functions of ACI.

Upon completion of the course, students will be familiar with:

  • Pre-Deployment Interview/Information gathering process
  • Out of the box setup and configuration of an ACI fabric
  • Scenario driven configuration principals for:
    • Multi-Tenancy/Tenant Configurations
    • Application Profiles
    • Endpoint Groups
    • Security - Built-in Stateful Firewall, Contracts
    • Virtual Machine Manager (VMM) Integration
    • Connecting the ACI fabric to external L2 and L3 networks
    • Connecting fabric to L4 – L7 services (ASA, F5, etc….)

Course Specifics

  • This course has a high degree of labs (40:60 ratio lab:lecture), and is intended to engage and challenge the student into performing the most common but also complex scenarios when migrating to Cisco Application Centric Infrastructure (ACI) Fabric
  • Hands on lab environments. Each student will have:
    • Dedicated ACI tenant with 4 ESXi hosts and 9 VM’s
    • Dedicated vCenter Appliance
    • Dedicated L2/L3 outside connections
  • Class size will be restricted to 16 students to facilitate more hands on lab time.
  • Each student will have a dedicated Tenant and dedicated ports both on the ACI Leafs and on the external Nexus 5K/6Ks, dedicated 4 ESXi servers, vCenter server and VMs.
  • Students do not share or “partner-up” to perform tasks. Each student independently performs all tasks.

Who Should Participate?

  • Solution Architects
  • System Integrators
  • Network Administrators
  • System Engineers

Recommended Prerequisites

  • Solid understanding of basic Layer 2 / Layer 3 networking concepts
  • Basic knowledge of virtualization concepts and technology
  • Understand basic firewall and load balancing principles
  • Basic Linux system administration (helpful, but not required)

    • Course Agenda

    Day 1
    Lecture 00 Introduction, Facilities and Agenda
    Lecture 01 Overview of the ACI Architecture
    Discuss the trends in the networking industry that got us to where we are today. Review the high level components that make up an ACI Fabric, and how they relate to and were formed by these trends. Outline the challenges that ACI has been created to solve in real world data centers.
    Lecture 02 Nexus 9000 Switch Architecture
    Review the physical switch competes that make up an ACI Fabric. This section ensures that students have a solid understanding of the hardware that underpins and works with the software that represents an ACI data center. Learn how the Nexus 9000 switch family differentiates itself from traditional Nexus switches.
    LUNCH
    Lecture 03 Application Policy Infrastructure Controller (APIC)
    This module not only covers the physical characteristics that make up the Application Policy Infrastructure Controller (APIC for short), but also begins to dive into the fundamentals of the managed information tree and how the APIC helps to logically model applications.
    Lab 01 Initial Fabric Discovery
    Day 2
    Lecture 04 Fabric Access - Physical Policies
    Learn about the basics of the internal ACI fabric configurations required to setup and operate an ACI data center. This lecture also dives into the logical constructs that dictate how an ACI Fabric treats the physical switch ports in the network.
    Lab 02 Fabric Access Policy Configuration
    LUNCH
    Lecture 05 Tenant - Logical Overlay Policies
    Tenancy in ACI is extremely powerful and can be leveraged to provide many benefits, learn about the logical model that defines tenancy as well as the components that make up a tenant. By the end of this module, you will have a solid understanding of the building blocks that make up ACI and the logical model.
    Lab 03 Initial Tenant Configuration
    Lab 04 App Profiles and EPGs
    Day 3
    Lecture 06 Fabric Operation and Forwarding
    Now that you understand (and have configured!) the logical components of ACI, its time to understand how all of this works under the covers. This module dives right into the forwarding fundamentals that drive ACI, with extra emphasis on VXLAN and how it is leveraged to make ACI do some of the incredible things it does. Expect to spend plenty of time white boarding how packets flow through the fabric!
    Lecture 07 Virtual Machine Management (VMM)
    By now you’ve seen how ACI handles connecting physical device to the fabric, but what about virtual servers/devices? This module covers that! Learn about how ACI supports any hypervisor out there from VMware to Microsoft, Openstack to Xen, even containers!
    LUNCH
    Lab 05 VMM Integration
    Lab 06 Filters, Subjects and Contracts
    Day 4
    Lecture 08 Bridging L2 to Existing Networks
    Discuss how ACI handles layer two connectivity outside of the fabric. Talk about design considerations around Spanning­Tree and Miscable Protocol. Review common deployment scenarios surrounding layer two extension from the fabric.
    Lab 07 L2 External Connectivity
    LUNCH
    Lecture 09 Routing L3 to External Networks
    Learn about how ACI handles layer three connectivity to external devices. Review how ACI functions as a transit domain, and all of the routing options configurable in the APIC. Finally, discuss how this is being done in production ACI fabrics and review design considerations for deploying this yourself.
    Lab 08 L3 External Connectivity
    Lab 09 Bringing it All Together - Inter-Tenant Routing, VM Migration
    Day 5
    Lecture 10 Planning and Capacity Sizing
    Here we will spend a few crucial minutes discussing how to plan for scale and capacity with the purpose-built tool, built directly into every Cisco ACI APIC controller (and ACI Simulator) using the “ACI Optimizer” tool, and how to monitor growth well into production using the “Capacity Dashboard” tool, amongst many others.
    Lab 10 ACI Optimizer
    LUNCH
    Lecture 11 Day 2 Operations and Troubleshooting
    Lab 11 Troubleshooting
