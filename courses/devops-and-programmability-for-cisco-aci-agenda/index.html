Cisco DevOps and Programmability for Cisco ACI and Nexus 9000

Offering Formats

This course is offered in the following formats:

  • Live, Instructor-Led Training (schedule)
  • Self-Paced, Online Training (samples)

Course Overview

In this 2-day course, you will learn how to begin programming and automating Cisco ACI and VXLAN fabrics. You will become experienced with the the implementation of Application Centric Infrastructure (ACI) entirely via REST and the Python native SDK – Cobra (with emphasis on the easy to use ACI Toolkit for consuming Cobra). You will also become experienced on what a true DevOps culture is, how it is formed, how a team functions in their new environment.

It is designed for customers, systems engineers, and field engineers who wanting to learn how to accelerate their deployments with RESTful API calls. Students will learn in labs not only how to use Postman for ACI, but how to create and store collections of csv-provded, variable-driven REST calls, that can be run all at once via Postman Collection Runner. With this, any customer environment can quite literally be deployed in seconds. Use of automation tools such as Ansible for ACI will be learned through hands-on labs. Hands-on labs will show users how to quickly adopt native Python SDK (Cobra) as well as even faster and easier Python adoption via the Cisco ACI Toolkit. Discussion will also be presented on how to bring about a change to a DevOps culture in your company.

Who Should Attend?

  • Network Administrators
  • System Engineers
  • Solution Architects
  • System Integrators

Recommended Prerequisites

  • Basic understanding of networking
  • Basic understanding of Nexus switches
  • Basic understanding of Cisco ACI (recommended, but not required)

Course Agenda

Day 1
Lecture 01 What is DevOps?
Why Should I Care About Programmability? Who Are We Talking To? How do we Break Down the Walls? Software Development Models, Software Architectural Models, Taking Baby Steps
Lab 00 Teaser Lab
Students will execute a simple Python Lecture and watch your entire ACI Fabric built in seconds. Brief test of VMs to ensure data plane functionality is as expected
Lab 01 Preparing Your Development Environment
Students work to take a base Ubuntu install and add the necessary applications and toolsets for automation and programming so they have an understanding in the real world, not just how to use, but where to obtain and how to install toolsets
LUNCH BREAK
Lecture 02 Programmability Concepts
What is an API? What is Python? What is REST? What is JSON? What is JSON-RPC? What is XML?
Lab 02 Getting to Know Git
Students learn to work with the GitHub Windows Desktop Client, then learn to work with the git CLI in Linux. Students create and push their first repository, as well as clone a number of repositories that we will use throughout our course
Lecture 03 ACI Programmability
How to interact with the ACI API REST API, Native Python Cobra SDK and ACI Toolkit
Lab 03 Capturing ACI REST API w/ API Inspector and Remote Logger
Students use both the ACI API Inspector and the Remote Logger to create a few simple Tenant children objects in order to learn how the API Inspector works, and how to use a Sublime Text to format and work with XML and JSON
Lab 04 Learning to Love the Postman and his Jetpacks
Students start by learning the basics of Postman, then move on to learning advanced Postman usage with Postman Interceptor, Jetpacks, Postman Collections, and how replace key fields in our captured REST API JSON data with variables and then use Google Sheets to create all desired values for as many iterations of an item they wish to create, then use Postman Collections Runner to automate everything. Also learn how Postman auto-generates Python, cURL, wget and even PHP to remotely execute your REST.
Day 2
Lab 05 Introduction to Basic Python Calling REST for ACI
Students automate a collection of REST API calls by creating a simple Python program that calls ‘requests’ and ‘json’ and adds the REST API calls all inline, and runs the Python program. Also brief discussion of RegEx and variable replacement
Lab 06 Auto-Generating Python for ACI Cobra
Students learn to use Arya and Remote Logger to auto-generate Python
Lab 07 ACI Toolkit Goodness
Students git clone the ACIToolkit and work first with the built-in applications, creating a mysql DB and using both the Endpoint Tracker as well as ACI Snapshot manager to compare configs and rollback to previous known good ACI deployments
LUNCH BREAK
Lab 08 Advanced Python Programming Using the ACI Toolkit
Students go much deeper in beginning to learn how to use the ACI Toolkit to make native Python programming with ACI much easier to understand and use through the use of the abstraction of the ACI Toolkit that Cisco has published and made publicly available
Lecture 04 Automation Toolsets
Students learn how to work with Ansible Playbooks, Puppet Manifests and Chef Recipes and Cookbooks, as well as the basics of CI/DI and Jenkins/Maven.
Lab 09 “Play”ing with Ansible and Automating ACI
Students git clone some Ansible Playbooks and learn about YAML and how to work with it to automate full ACI deployments
Lecture 05 Introduction to NX-OS Programability
Students learn about Guest Shell, Bash and root access in new Open NX-OS (Camden), RPM/YUM installation, how third part apps can bring added value, as well as learn about Model Driven Programmability (MDP) and Visore access to N9K.
